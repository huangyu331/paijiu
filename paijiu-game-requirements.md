# 牌九棋牌游戏 - 产品需求文档

## 1. 产品概述

### 1.1 产品名称
**牌九大师** (Pai Gow Master)

### 1.2 产品定位
一款界面简洁美观、功能齐全的**纯前端单机版**牌九棋牌游戏，使用传统 32 张骨牌，无需后端服务器，开箱即用。

### 1.3 目标用户
- 喜欢传统中国棋牌游戏的玩家
- 想学习牌九规则的新手
- 休闲娱乐用户
- 希望离线游玩的用户

### 1.4 技术特点
- **纯前端实现** - 无需服务器，浏览器直接运行
- **本地存储** - 使用 localStorage 保存游戏进度和数据
- **离线可用** - 支持 PWA，可离线游玩
- **跨平台** - 支持 PC、手机、平板浏览器

#### 2.3.3 非对牌（点数比较）

若无对牌，则以**两张牌点数之和的个位数**分胜负：

| 点数 | 名称 | 说明 |
|------|------|------|
| 9 点 | 至尊九 | 最大 |
| 8 点 | 八点 | - |
| 7 点 | 七点 | - |
| 6 点 | 六点 | - |
| 5 点 | 五点 | - |
| 4 点 | 四点 | - |
| 3 点 | 三点 | - |
| 2 点 | 二点 | - |
| 1 点 | 一点 | - |
| 0 点 | 瘪十 | 最小（如 10+10=20→0） |

**点数计算示例：**
- 天牌 (12) + 杂五 (5) = 17 → **7 点**
- 梅花 (10) + 红头 (10) = 20 → **0 点**（瘪十）
- 天牌 (12) + 8 点牌 = 20 → **0 点**（天杠）

### 2.4 比牌规则

#### 2.4.1 基本规则
1. **对牌 > 非对牌**（任何对牌都大于没对牌）
2. **同对牌比较**：按对牌大小顺序
3. **同点数比较**：点数相同时，比较最大单张牌的级别
4. **完全相同**：庄家赢（庄家优势）


#### 2.4.2 小牌九规则
- 每人 2 张牌，直接与庄家比
- **胜**：赢 1 倍下注
- **和**：庄家赢（庄家优势）
- **负**：输掉下注

---

### 2.5 游戏流程

1. **下注** - 玩家选择下注金额
2. **砌牌** - 庄家将 32 张牌面朝下砌成 8 排，每排 4 张
3. **掷骰** - 庄家掷骰子决定派牌起始位置
4. **派牌** - 按逆时针方向派牌，每家 2 张
6. **比牌** - 与庄家比牌
7. **结算** - 根据胜负结算筹码

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 游戏模式
| 模式 | 描述 |
|------|------|
| 小牌九（推荐） | 2 张牌直接比，快节奏 |
| 挑战模式 | 有限筹码（10000 起始），冲击高分 |
| 无尽模式 | 筹码不重置，持续挑战直到输光 |
| 教学关卡 | 逐步学习规则和牌型 |

#### 3.1.2 游戏操作
- **下注** - 选择筹码数量（支持自定义）
- **砌牌动画** - 展示 32 张牌砌成 8 排
- **掷骰动画** - 骰子决定派牌位置
- **派牌动画** - 按规则派发到各门
- **比牌** - 自动与庄家比牌
- **结算** - 显示胜负结果和筹码变化

#### 3.1.3 辅助功能
- **规则说明** - 详细游戏规则和牌型介绍
- **牌型查询** - 随时查看牌型大小顺序
- **新手引导** - 首次游戏的教学引导
- **历史记录** - 查看最近游戏记录

### 3.2 用户系统（本地）

#### 3.2.1 账户功能
- **本地账户** - 无需注册，浏览器自动识别
- **多存档支持** - 可创建多个本地存档
- **个人资料** - 昵称、头像（本地上传）
- **游戏统计** - 胜率、总局数、最高连胜、总盈利

#### 3.2.2 筹码系统
- **初始筹码** - 新游戏赠送 10,000
- **每日奖励** - 每日首次登录赠送 1,000
- **成就奖励** - 完成成就获得筹码
- **破产保护** - 筹码归零后可免费重置（每日 1 次）
- **数据持久化** - 所有数据保存在 localStorage

### 3.3 规则设置（可选）

| 规则 | 选项 | 说明 |
|------|------|------|
| 特殊组合 | 开启 / 关闭 | 天王、地王、天杠、地杠等 |
| 庄家优势 | 开启 / 关闭 | 平局时庄家赢 |

## 5. 技术需求

### 5.1 技术栈建议

#### 前端框架（推荐方案）
| 方案 | 技术栈 | 优势 |
|------|--------|------|
| **方案 A** | Vue 3 + Vite + Pinia | 轻量、易上手、中文文档完善 |
| **方案 B** | React + Vite + Zustand | 生态丰富、组件库多 |
| **方案 C** | 原生 HTML/CSS/JS | 零依赖、体积最小 |

#### UI 组件库
- **Naive UI** (Vue 3) - 主题定制方便，组件美观
- **Ant Design Vue** - 企业级组件，稳定可靠
- **Tailwind CSS** - 原子化 CSS，灵活定制

#### 动画库
- **GSAP** - 强大的动画时间轴控制
- **Framer Motion** (React) - 声明式动画
- **CSS Animations** - 简单动画用 CSS 即可

#### 数据存储
- **localStorage** - 主要数据存储
- **IndexedDB** - 大量数据时使用
- **PWA Cache** - 离线资源缓存

### 5.2 性能要求
- **首屏加载** < 2 秒（本地资源）
- **动画帧率** ≥ 60fps
- **包体积** < 5MB（gzip 后）
- **操作响应** < 50ms
- **内存占用** < 100MB

### 5.3 安全与隐私
- **本地存储** - 所有数据保存在用户浏览器
- **无需登录** - 不收集用户个人信息
- **随机数** - 使用 crypto.getRandomValues() 保证公平
- **数据导出** - 支持导出/导入存档数据
- **隐私保护** - 不追踪、不上传任何数据
