# 牌九棋牌游戏 - 产品需求文档

## 1. 产品概述

### 1.1 产品名称
**牌九大师** (Pai Gow Master)

### 1.2 产品定位
一款界面简洁美观、功能齐全的**纯前端单机版**牌九棋牌游戏，无需后端服务器，开箱即用。

### 1.3 目标用户
- 喜欢传统中国棋牌游戏的玩家
- 想学习牌九规则的新手
- 休闲娱乐用户
- 希望离线游玩的用户

### 1.4 技术特点
- **纯前端实现** - 无需服务器，浏览器直接运行
- **本地存储** - 使用 localStorage 保存游戏进度和数据
- **离线可用** - 支持 PWA，可离线游玩
- **跨平台** - 支持 PC、手机、平板浏览器

## 2. 游戏规则

### 2.1 基本规则

#### 2.1.1 牌具
- 使用 32 张牌九牌（天九牌）
- 每张牌由两个骰子点数组合而成
- 牌面点数范围：1-6 点

#### 2.1.2 牌型大小（从大到小）

**特殊牌型：**
1. **至尊宝** - 丁三 + 二四（最大牌）
2. **天王** - 天牌 + 天牌
3. **地王** - 地牌 + 地牌

**对子（按点数）：**
- 天牌对（6-6）
- 地牌对（1-1）
- 人牌对（4-4）
- 和牌对（1-3）
- 梅花对（5-5）
- 长三对（3-3）
- 板凳对（2-2）
- 斧头对（5-1）
- 红头十对（4-6）
- 高脚七对（1-6）
- 零霖六对（3-6）
- 杂五对（5-2）
- 杂六对（2-6）
- 杂七对（1-5）
- 杂八对（2-4）
- 杂九对（3-5）
- 杂十对（4-5）
- 杂十一对（5-6）
- 杂十二对（6-5）

**非对子（比较两张牌点数之和）：**
- 点数和大的赢
- 点数和相同则比较单张最大牌

### 2.2 游戏流程

1. **下注阶段** - 玩家选择下注金额
2. **发牌阶段** - 庄家给每位玩家发 4 张牌
3. **分牌阶段** - 玩家将 4 张牌分成前后两组（每组 2 张）
4. **比牌阶段** - 前后两组分别与庄家比牌
5. **结算阶段** - 根据胜负结算筹码

### 2.3 胜负判定

- **前组胜 + 后组胜** = 双赢（赢得双倍下注）
- **前组胜 + 后组负** 或 **前组负 + 后组胜** = 和局（退还下注）
- **前组负 + 后组负** = 双输（输掉下注）

**分牌规则：**
- 前组（小牌）必须 ≤ 后组（大牌）
- 违反规则视为"犯规"，直接判负

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 游戏模式
| 模式 | 描述 |
|------|------|
| 快速游戏 | 即开即玩，无限筹码，适合新手练习 |
| 挑战模式 | 有限筹码（10000 起始），冲击高分 |
| 无尽模式 | 筹码不重置，持续挑战直到输光 |
| 教学关卡 | 逐步学习规则和分牌技巧 |

#### 3.1.2 游戏操作
- **下注** - 选择筹码数量（支持自定义）
- **发牌** - 庄家发牌动画
- **分牌** - 拖拽或点击方式分组 4 张牌
- **自动分牌** - AI 辅助最优分牌建议
- **比牌** - 自动与庄家比牌
- **结算** - 显示胜负结果和筹码变化

#### 3.1.3 辅助功能
- **规则说明** - 详细游戏规则和牌型介绍
- **牌型查询** - 随时查看牌型大小顺序
- **新手引导** - 首次游戏的教学引导
- **分牌提示** - 显示推荐分牌方案
- **历史记录** - 查看最近游戏记录

### 3.2 用户系统（本地）

#### 3.2.1 账户功能
- **本地账户** - 无需注册，浏览器自动识别
- **多账户支持** - 可创建多个本地存档
- **个人资料** - 昵称、头像（本地上传）
- **游戏统计** - 胜率、总局数、最高连胜、总盈利

#### 3.2.2 筹码系统
- **初始筹码** - 新游戏赠送 10,000
- **每日奖励** - 每日首次登录赠送 1,000
- **成就奖励** - 完成成就获得筹码
- **破产保护** - 筹码归零后可免费重置（每日 1 次）
- **数据持久化** - 所有数据保存在 localStorage

## 5. 技术需求

### 5.1 技术栈建议

#### 前端框架（推荐方案）
| 方案 | 技术栈 | 优势 |
|------|--------|------|
| **方案 A** | Vue 3 + Vite + Pinia | 轻量、易上手、中文文档完善 |
| **方案 B** | React + Vite + Zustand | 生态丰富、组件库多 |
| **方案 C** | 原生 HTML/CSS/JS | 零依赖、体积最小 |

#### UI 组件库
- **Naive UI** (Vue 3) - 主题定制方便，组件美观
- **Ant Design Vue** - 企业级组件，稳定可靠
- **Tailwind CSS** - 原子化 CSS，灵活定制

#### 动画库
- **GSAP** - 强大的动画时间轴控制
- **Framer Motion** (React) - 声明式动画
- **CSS Animations** - 简单动画用 CSS 即可

#### 数据存储
- **localStorage** - 主要数据存储
- **IndexedDB** - 大量数据时使用
- **PWA Cache** - 离线资源缓存

### 5.2 性能要求
- **首屏加载** < 2 秒（本地资源）
- **动画帧率** ≥ 60fps
- **包体积** < 5MB（gzip 后）
- **操作响应** < 50ms
- **内存占用** < 100MB

### 5.3 安全与隐私
- **本地存储** - 所有数据保存在用户浏览器
- **无需登录** - 不收集用户个人信息
- **随机数** - 使用 crypto.getRandomValues() 保证公平
- **数据导出** - 支持导出/导入存档数据
- **隐私保护** - 不追踪、不上传任何数据

### 8.2 术语表
- **庄家** - 银行家，玩家与之比牌
- **分牌** - 将 4 张牌分成前后两组
